### The production deploy and the live verification (§8; the human's part)

- The AI prepares; the human runs: `npx sst secret set OerApiKey <key> --stage production` → `npx sst deploy --stage production` → the verification sweep against the live URL:
  - `GET /health` — 200, the version 0.8.0, `ratesCacheAge` null before the first conversion;
  - `GET /api/init` — 200 with the ETag; a second request with `If-None-Match` → 304, empty body;
  - `GET /api/currencies` — 200, `Cache-Control: public, max-age=3600`, a real intersection (THE FIRST LIVE OER CALL — proves the secret);
  - `POST /api/convert` happy path — 200 with a real rate; the validation catalog spot-checks: a 3-decimal amount → 400 `amountTooManyDecimals`, `from = to` → 400 `sameCurrency`, an unknown currency → 422 with `params.code`;
  - `GET /api/stats` — the conversion above is IN the totals (proves the production DynamoDB write);
  - `GET /docs` 200 AND `GET /docs/static/swagger-ui.css` 200 (proves the copyFiles assets — the v0.7.0 finding);
  - an unknown route → the unified 404.
- The live URL lands in the README only after it exists — the README is finished in the same branch once the deploy succeeds (rule 19: main's README never describes non-existent functionality).
